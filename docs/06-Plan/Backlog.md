## Epic 1: 商品管理

> **Epic描述**：实现平台商品的全生命周期管理，包括商品分类、商品信息的创建、编辑、上下架等功能，为门店商品管理提供基础数据支撑。

### Story 1.1: 商品分类管理

| 字段 | 内容 |
|------|------|
| **Story ID** | US-1.1 |
| **优先级** | P0 (MVP必做) |
| **角色** | 系统管理员 |
| **描述** | 作为系统管理员，我希望能够管理商品分类，以便对平台商品进行分类组织，方便消费者浏览和筛选商品。 |

**验收标准（AC）**：
- [ ] AC1: 系统管理员可以创建新的商品分类，包含分类名称、图标、排序权重
- [ ] AC2: 系统管理员可以编辑已有分类的名称、图标、排序权重
- [ ] AC3: 系统管理员可以启用/禁用分类，禁用后该分类下的商品在消费者端不展示
- [ ] AC4: 分类列表按排序权重降序展示，支持拖拽调整顺序
- [ ] AC5: 删除分类时，如果分类下有商品，需提示"该分类下有X个商品，请先移除或转移商品"

**验收标准（AC）**：

## Epic 2: 门店库存价格管理

> **Epic描述**：实现门店对商品的独立库存和价格管理，支持门店从平台商品库选择商品上架，并设置本店的销售价格和库存数量。

### Story 2.1: 门店商品上架

| 字段 | 内容 |
|------|------|
| **Story ID** | US-2.1 |
| **优先级** | P0 (MVP必做) |
| **角色** | 门店管理员 |
| **描述** | 作为门店管理员，我希望能够从平台商品库中选择商品上架到本店，以便消费者可以在本店购买这些商品。 |

**验收标准（AC）**：
- [ ] AC1: 门店管理员可以浏览平台商品库，查看所有已启用的平台商品
- [ ] AC2: 门店管理员可以选择商品上架到本店，上架时必须设置本店销售价格和初始库存
- [ ] AC3: 上架时销售价格不得低于平台设定的最低限价（如有），否则提示错误
- [ ] AC4: 上架成功后，商品在本店消费者端可见
- [ ] AC5: 已上架的商品在平台商品库中显示"已上架"标识

**验收标准（AC）**：

## Epic 3: 订单履约

> **Epic描述**：实现消费者下单购买和门店订单处理的完整流程，包括购物车、订单创建、支付、门店接单、备货等环节。

### Story 3.1: 购物车管理

| 字段 | 内容 |
|------|------|
| **Story ID** | US-3.1 |
| **优先级** | P0 (MVP必做) |
| **角色** | 消费者 |
| **描述** | 作为消费者，我希望能够将商品加入购物车并管理购物车，以便一次性购买多个商品。 |

**验收标准（AC）**：
- [ ] AC1: 消费者可以将商品加入购物车，默认数量为1
- [ ] AC2: 消费者可以在购物车中修改商品数量（1-99）
- [ ] AC3: 消费者可以删除购物车中的商品
- [ ] AC4: 购物车与当前选择的门店绑定，切换门店时提示"切换门店将清空购物车"
- [ ] AC5: 购物车中显示商品的实时价格（随门店价格变动更新）
- [ ] AC6: 购物车中显示商品的库存状态，库存不足时显示"库存不足，仅剩X件"
